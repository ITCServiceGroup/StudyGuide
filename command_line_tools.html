Command-Line Tools Study Guide

1. Accessing Command-Line Interfaces

Different operating systems provide various ways to access their command-line interfaces. Understanding these access methods is crucial for network diagnostics.

Windows

  • Press Windows + R, type "cmd", press Enter
  • Search for "Command Prompt" in Start menu
  • Navigate to C:\Windows\System32 and run cmd.exe

macOS

  • Open Spotlight (Command + Space), type "Terminal"
  • Find Terminal in Applications > Utilities
  • Use Launchpad to locate Terminal
  • Use Network Utility:
    • Open Spotlight, type "Network Utility"
    • Find in Applications > Utilities folder
    • Access comprehensive network tools

Chrome OS

  • Press Ctrl + Alt + T to open Crosh shell
  • Use Linux Terminal (if enabled)
Important: Some commands may require administrator privileges. On Windows, right-click Command Prompt and select "Run as administrator". On macOS/Linux, use "sudo" before commands requiring elevated privileges.

Maps the path between your device and a destination:

Windows: tracert google.com

Packet Loss Testing

Send multiple pings to test connection stability:

Windows: ping -n 50 8.8.8.8
Reply from 8.8.8.8: bytes=32 time=15ms TTL=57 Reply from 8.8.8.8: bytes=32 time=14ms TTL=57 Packets: Sent = 50, Received = 50, Lost = 0 (0% loss)

Finding IP Addresses on Mac

Different network interfaces require different commands:

WiFi Connection: ipconfig getifaddr en0
192.168.1.100
Ethernet Connection: ipconfig getifaddr en1
192.168.1.101
All Network Interfaces: ifconfig
en0: flags=8863 inet 192.168.1.100 netmask 0xffffff00 broadcast 192.168.1.255 en1: flags=8863 inet 192.168.1.101 netmask 0xffffff00 broadcast 192.168.1.255
Pro Tip: Use "ifconfig | grep inet" to filter and only show IP addresses.
Pro Tip: When testing for packet loss, always send at least 50 packets to get a meaningful sample size. This helps identify intermittent issues that might not show up in a standard ping test.

4. DNS Cache Management

DNS cache can become corrupted or outdated, leading to connectivity issues. Knowing how to clear the DNS cache is essential for troubleshooting.

DNS Cache Clearing Commands:

Operating System Command Notes
Windows ipconfig /flushdns Requires admin privileges
macOS sudo killall -HUP mDNSResponder Password required
Chrome OS Clear browser cache DNS managed by Chrome
When to Clear DNS Cache:
  • After DNS server changes
  • When websites don't load correctly
  • After network configuration changes
  • If security issues are suspected

5. Interpreting Command Output

Understanding command output is crucial for effective troubleshooting.

Common Output Patterns:

  • Request timed out - No response received, indicates connection issues
  • Destination host unreachable - No route to target
  • TTL expired in transit - Packet took too many hops
  • General failure - Network adapter issues

Ping Results Interpretation:

Pinging 8.8.8.8 with 32 bytes of data:
Reply from 8.8.8.8: bytes=32 time=15ms TTL=57    ← Good response
Request timed out.                                ← Connection issue
Reply from 192.168.1.1: Destination unreachable   ← Routing problem
General failure.                                  ← Hardware/driver issue

Performance Metrics:

  • Response Time: < 50ms is excellent, > 150ms may indicate issues
  • Packet Loss: < 1% is normal, > 5% indicates problems
  • TTL Values: Typically 64 or 128, lower values show longer paths
  • Jitter: Variation in response times, should be minimal

6. Best Practices

Command-Line Troubleshooting Guidelines:

  1. Start with basic connectivity tests (ping)
  2. Progress to more detailed tests (traceroute)
  3. Document all command outputs
  4. Test from multiple locations when possible
  5. Use appropriate sample sizes for tests
Documentation Tip: When documenting command outputs, include:
  • Date and time of test
  • Complete command used
  • Full output received
  • Network conditions at time of test
